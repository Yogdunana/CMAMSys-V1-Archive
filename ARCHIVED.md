## 📌 归档原因

CMAMSys (CompetiMath AutoModel System) 是一个企业级数学建模竞赛自动化平台的 V1 版本。由于以下原因，本项目已停止开发并归档：

1. ✅ **项目目标已完成** - V1 版本功能已全部实现并交付
2. 🔄 **架构演进** - 项目将迁移到新的架构和技术栈
3. 📚 **学习参考价值** - 代码和文档对学习现代全栈开发具有参考价值

## 📖 使用须知

### 仅限学习参考

本仓库**仅用于学习和参考目的**，**请勿直接用于生产环境**。

### 安全警告

⚠️ **重要安全提示**：

1. **硬编码配置** - 代码中可能包含测试用的硬编码配置（数据库密码、API密钥等）
2. **默认凭证** - 绝不要在生产环境中使用默认账户和密码
3. **依赖漏洞** - 依赖包未更新，可能包含已知的安全漏洞
4. **缺少审计** - 代码未经过生产级别的安全审计

### 生产使用建议

如果你参考本项目的代码开发自己的系统，请务必：

1. ✅ 替换所有硬编码的敏感信息（密码、密钥、Token）
2. ✅ 更新所有依赖包到最新版本
3. ✅ 进行全面的安全审计和渗透测试
4. ✅ 配置适当的生产环境监控和日志
5. ✅ 实施完整的备份和灾难恢复方案

## 📚 技术栈参考

本项目使用了以下技术栈（归档时的版本）：

- **前端**: Next.js 16, React 19, TypeScript 5, Tailwind CSS 4
- **UI 组件**: shadcn/ui (Radix UI)
- **后端**: Next.js API Routes
- **数据库**: PostgreSQL 16
- **ORM**: Prisma 6
- **认证**: JWT + Refresh Tokens
- **包管理**: pnpm
- **容器化**: Docker + Docker Compose

### 核心功能

- 🔐 用户认证（邮箱、MFA、密码重置）
- 👥 用户管理和权限控制
- 📊 竞赛管理
- 🤖 AI Provider 集成（阿里云百炼、火山引擎、DeepSeek）
- 📝 论文管理和在线编辑
- 📈 数据分析和可视化
- 🔍 审计日志
- 🌐 国际化（i18n）
