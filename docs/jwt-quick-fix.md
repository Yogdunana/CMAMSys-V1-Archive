# 快速修复 JWT 签名验证失败

## 立即解决（30秒）

### 方法 1：使用浏览器控制台（最快）

1. 在当前页面按 **F12** 打开开发者工具
2. 点击 **Console** 标签
3. 复制粘贴以下代码并按回车：

```javascript
localStorage.clear();
location.reload();
```

### 方法 2：使用退出登录按钮

1. 点击页面右上角的用户头像/菜单
2. 点击 "退出登录"
3. 重新登录

### 方法 3：清除浏览器数据

1. 按 **Ctrl + Shift + Delete** (Windows) 或 **Cmd + Shift + Delete** (Mac)
2. 选择 "Cookies 和其他网站数据"
3. 选择时间范围："全部时间"
4. 点击 "清除数据"
5. 刷新页面并重新登录

## 为什么会出现这个问题？

这种情况通常发生在：
- 系统更新后 JWT 密钥发生变化
- 浏览器中存储了旧的登录凭证
- 在不同环境之间切换（开发/测试/生产）

## 预防措施

为避免此问题再次出现：
- 定期退出登录（每周一次）
- 不要在多个浏览器/设备上同时登录
- 清除浏览器缓存后再登录

## 如果问题仍然存在

如果以上方法都无法解决问题，请联系系统管理员并提供以下信息：
- 使用的浏览器及版本
- 错误发生的具体步骤
- 浏览器控制台的错误信息
