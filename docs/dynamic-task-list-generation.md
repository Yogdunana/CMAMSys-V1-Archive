# 动态任务列表生成功能说明

## 问题背景

之前系统中任务列表是硬编码在前端的，针对第一次问题的特定任务（遗传算法、蚁群算法等），无法根据不同题目类型和讨论结果动态调整。

## 解决方案

### 1. 讨论提示词增强

在 `src/services/group-discussion.ts` 中：

#### 新增规则
- 在第二轮讨论中，AI 必须输出【任务列表】部分
- 任务列表应包含 5-8 个系统化的建模任务
- 任务顺序应体现建模的逻辑流程：
  - 数据预处理和分析
  - 核心算法/模型构建
  - 求解/优化
  - 结果验证
  - 可视化和报告

#### 示例格式
```
【任务列表】
1. 分析电力需求数据和发电站特性，建立数据预处理流程
2. 构建多目标优化数学模型，包含成本和碳排放约束
3. 实现遗传算法基础框架（种群初始化、选择、交叉、变异）
4. 实现粒子群优化算法，与遗传算法混合
5. 设计求解算法，处理24小时时序优化问题
6. 进行结果验证，对比不同调度方案的优劣
7. 生成电力调度可视化报告，展示24小时调度结果
```

### 2. 讨论总结提取

在 `completeDiscussion` 函数中：

- 从第二轮讨论消息中提取【任务列表】
- 将任务列表合并并保存到 `discussion.summary.taskList`

### 3. 前端动态显示

在 `src/app/dashboard/auto-modeling/[id]/page.tsx` 中：

- 从 `discussionSummary.taskList` 或 `discussion.summary.taskList` 读取任务列表
- 根据任务列表动态生成 TODO 列表
- 如果没有任务列表，使用默认列表兜底

### 4. 接口更新

- `TaskStatus` 接口新增 `discussionSummary` 字段
- 支持从两个来源读取任务列表：
  1. `taskStatus.discussionSummary.taskList`
  2. `taskStatus.discussion.summary.taskList`

## 效果

1. ✅ 任务列表根据讨论结果动态生成
2. ✅ 任务顺序体现建模的逻辑流程
3. ✅ 适应不同类型的数学建模问题
4. ✅ 提供默认列表兜底，确保系统稳定

## 使用指南

### 创建新任务时

1. 提交赛题后，系统自动启动讨论流程
2. 第二轮讨论中，AI 会生成【任务列表】
3. 讨论完成后，任务列表自动保存
4. 前端页面显示动态任务列表

### 查看任务列表

在任务详情页面的"任务进度"标签页，可以看到：
- 根据讨论结果生成的任务列表
- 当前任务完成状态
- 任务执行进度

### 注意事项

1. 任务列表的准确性取决于 AI 的讨论质量
2. 如果 AI 未生成任务列表，系统会使用默认列表兜底
3. 任务列表数量建议在 5-8 个之间
4. 任务描述应简洁明了，便于理解
